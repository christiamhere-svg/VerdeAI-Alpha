# Browser and Testing Limitations — v9.3.2

## Automated coverage completed

The included Chromium harness validates:

- build identification;
- photo opacity and filter state;
- six-future visibility and selection;
- procedural botanical geometry;
- plant-type and variant diversity;
- contact-shadow presence;
- distinct Wildlife, Feature, and Low-Maintenance signatures;
- correct Feature Garden wording;
- calibration completion;
- marker-5 visibility;
- AI safety lock;
- zero horizontal overflow at 360, 430, 768, and 1440 px.

## What automation cannot prove

- Whether botanical silhouettes look convincing on every real property photo.
- Whether a user interprets scale and perspective as believable.
- Whether calibration is comfortable on a particular Android device.
- Whether plants avoid every real object without manual keep-clear calibration.
- Whether physical refresh persistence behaves identically across all mobile browsers.

## Evidence photograph

Automated screenshots use a clean crop of the real overgrown-property photograph supplied during Android testing. This avoids app-interface text being baked into the test image.

## Release wording

The package is deployment-ready, but tester-ready status remains pending physical Android approval.
