# Android Realism Test Checklist — v9.3.2

Use a clean original property photograph, not a screenshot containing app or social-media controls.

## 1. Confirm the build
- [ ] Badge reads `Public Beta · Build v9.3.2`.
- [ ] Force-refresh if an older build appears.

## 2. Create a result
- [ ] Upload one clean property photograph.
- [ ] Open Photo and clue settings.
- [ ] Choose the closest situation and preferred direction.
- [ ] Run the analysis.

## 3. Check the main photograph
- [ ] The original property remains clearly recognisable.
- [ ] There is no heavy full-image green wash.
- [ ] Plants remain inside the photograph.
- [ ] Plants do not cover the whole image.
- [ ] Plants appear connected to the ground through shadows or darker bases.
- [ ] Foreground plants look larger and more detailed than rear plants.
- [ ] The access route remains visibly open.

## 4. Compare futures
- [ ] Select Wildlife Haven.
- [ ] Select Feature Garden.
- [ ] Select Low-Maintenance Haven.
- [ ] Wildlife Haven appears denser and more varied.
- [ ] Feature Garden has one clear focal composition and more open ground.
- [ ] Low-Maintenance Haven uses fewer plant families and stronger negative space.
- [ ] The selected-future label changes immediately.
- [ ] Recommended and Selected remain independent.

## 5. Check text
- [ ] Feature Garden explanation does not mention Recovery Garden.
- [ ] Title, composition, marker-5 move, report, and export all match the selected future.

## 6. Check calibration
- [ ] Help VerdeAI place the concept opens.
- [ ] Ground points drag correctly.
- [ ] Keep-clear boxes protect structures.
- [ ] Access handles protect the route.
- [ ] Opportunity and marker 5 move correctly.
- [ ] Done placing concept hides editor handles.
- [ ] Marker 5 remains visible.
- [ ] Refresh preserves the completed placement.

## 7. Final judgement
Answer honestly:

**Can I picture how planting would occupy this real property, without the overlay looking like simple cartoon icons?**

- [ ] Yes — realism gate passed.
- [ ] No — capture screenshots and keep the build out of tester release.
