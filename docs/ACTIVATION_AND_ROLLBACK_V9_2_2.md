# VerdeAI v9.2.2 — Activation and Rollback

## Activation sequence

Do not deploy the frontend first. The Worker must be deployed and verified before its URL is inserted into VerdeAI.

### 1. Prepare the Worker

From the separate Worker package:

```bash
npm install
npm run codes
```

`npm run codes` creates `pilot-codes-owner-only.txt`. Keep this file private and out of Git. It contains ten tester codes plus the comma-separated SHA-256 hashes required by the Worker.

### 2. Add the three Worker secrets

```bash
npx wrangler secret put OPENAI_API_KEY
npx wrangler secret put RATE_LIMIT_SALT
npx wrangler secret put PILOT_INVITE_CODE_HASHES
```

Use a long random value for `RATE_LIMIT_SALT`. For `PILOT_INVITE_CODE_HASHES`, paste only the comma-separated hashes generated by the script.

### 3. Deploy the approved Worker configuration

```bash
npx wrangler deploy
```

Copy the resulting HTTPS Worker URL.

### 4. Verify the Worker health response

Open:

```text
https://YOUR-WORKER.workers.dev/api/health
```

Before the frontend is connected, verify:

- `version` is `9.2.2`
- `realRenderingEnabled` is `true`
- `killSwitchOn` is `false`
- `testModeOn` is `false`
- `providerKeyPresent` is `true`
- `inviteCodesConfigured` is `true`
- `retentionPolicy` is `no-store`
- `pilotSpendCapUsd` is `5`
- `invitedTesterLimit` is `10`
- `maxCostPerRenderUsd` is `0.15`

The response must never reveal secret values.

### 5. Insert the Worker URL into the frontend

From the frontend package root:

```bash
node scripts/configure-worker-url.mjs https://YOUR-WORKER.workers.dev
```

This updates both active config aliases. Re-run validation before deploying to Cloudflare Pages.

### 6. Deploy the frontend

Deploy the contents of the frontend folder to the existing `verdeai-alpha.pages.dev` Pages project. Then conduct one owner-only real-photo test. Do not distribute tester codes until that owner test passes.

## Emergency rollback

The fastest server-side stop is:

```bash
npx wrangler deploy --config wrangler.safe-lock.jsonc
```

The safe-lock configuration changes the server state to:

- Real rendering disabled
- Kill switch on
- Test mode on
- Pilot spend cap zero
- Tester limit zero

The free calibrated overlay remains available because it does not depend on the Worker.

The frontend button **Stop AI calls on this browser** is only a local browser lock. It cannot replace the server-side rollback.

## Strong warning

Do not place the OpenAI API key, rate-limit salt, raw invite codes or invite-code hashes in `index.html`, JavaScript, GitHub, Cloudflare Pages variables, screenshots or tester messages.
