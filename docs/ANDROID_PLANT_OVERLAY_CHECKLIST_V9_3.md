# VerdeAI v9.3 — Android Plant Overlay Checklist

Use a real property photograph. Test at least one shaded/under-building area and one open yard.

## Photograph and calibration

- [ ] The uploaded photograph is the dominant result surface.
- [ ] Ground points 1–4 remain easy to drag.
- [ ] Keep-clear boxes can cover roofs, walls, doors, windows, furniture or machinery.
- [ ] A1 and A2 preserve the real access line.
- [ ] Opportunity point O moves the main visual emphasis.
- [ ] Marker 5 moves the first reversible test.
- [ ] No horizontal page overflow appears while editing.

## Finished concept

Tap **Done placing concept**, then verify:

- [ ] all editor handles and outlines disappear
- [ ] marker 5 remains visible
- [ ] plants remain aligned to the photograph
- [ ] plants do not cover protected structures
- [ ] the access route remains visually open
- [ ] foreground, middle and rear planting are distinguishable
- [ ] the overlay still reads clearly without zooming

## Future differentiation

- [ ] Belonging Garden has a clear focal plant and framed approach.
- [ ] Sanctuary Garden looks restrained, repeated and low-care.
- [ ] Possibility Garden looks layered, varied and habitat-rich.
- [ ] Gathering Space visibly preserves an open social zone.
- [ ] Productive Garden shows beds/herbs rather than ordinary shrubs.
- [ ] Maker / Workshop Yard protects work and storage zones.

## Persistence

- [ ] Refresh the page after finishing placement.
- [ ] Calibration remains aligned.
- [ ] Marker 5 remains in the chosen position.
- [ ] The selected future remains selected.
- [ ] Recommended and selected futures remain independent.

## Hard release gate

Ask without explaining the app:

> Can you picture how plants would occupy this exact space?

- [ ] **Yes immediately:** candidate pass.
- [ ] **Only after explanation:** fail and revise.
- [ ] **No:** fail and revise.
