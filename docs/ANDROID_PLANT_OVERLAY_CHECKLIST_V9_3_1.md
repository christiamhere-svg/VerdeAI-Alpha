# VerdeAI v9.3.1 — Android Plant Overlay Test Checklist

Use the deployed live site and a real property photograph that you have permission to use. Force-refresh before testing.

## A. Build and photo clarity

- [ ] Header shows **Public Beta · Build v9.3.1**.
- [ ] Upload a real property photograph.
- [ ] Complete analysis and open the result.
- [ ] Building, roofline, paving and existing ground remain clearly recognisable.
- [ ] There is no broad green wash over the whole photograph.
- [ ] Planting appears only in intended ground areas.

## B. Calibration preservation

- [ ] Open **Help VerdeAI place the concept**.
- [ ] Drag the usable-ground handles.
- [ ] Add or adjust a keep-clear zone.
- [ ] Adjust the protected access route.
- [ ] Adjust the opportunity point and marker 5.
- [ ] Press **Done placing concept**.
- [ ] All editor handles and outlines disappear.
- [ ] Marker 5 remains visible.
- [ ] Plants avoid the building, keep-clear region and protected route.

## C. Six-future access

Scroll below the recommended result.

- [ ] **Other possible futures** is visible before Optional AI Rendering.
- [ ] Exactly six future cards are present.
- [ ] Feature Garden is reachable.
- [ ] Low-Maintenance Haven is reachable.
- [ ] Wildlife Haven is reachable.
- [ ] Gathering Space is reachable.
- [ ] Food Garden is reachable.
- [ ] Maker / Workshop Yard is reachable.
- [ ] No horizontal scrolling is required.

## D. Selection behaviour

- [ ] Select Wildlife Haven.
- [ ] The main property photograph updates immediately.
- [ ] Wildlife Haven is clearly marked as selected.
- [ ] VerdeAI’s recommended future does not silently change.
- [ ] Return to VerdeAI Concept and confirm the recommendation still displays correctly.
- [ ] Return to Selected Future and confirm Wildlife Haven is retained.

## E. Visual differentiation

- [ ] Gathering Space keeps an obvious open social centre.
- [ ] Wildlife Haven is visibly denser, more organic and more varied.
- [ ] Low-Maintenance Haven uses repeated restrained planting and more open space.
- [ ] Food Garden shows recognisable productive beds or edible planting.
- [ ] Feature Garden has one dominant focal composition.
- [ ] Maker / Workshop Yard preserves practical access and working space.
- [ ] The six options do not look like the same overlay recoloured.

## F. Grounding and readability

- [ ] Foreground, middle and rear plant layers are visible.
- [ ] Plant bases appear connected to the ground rather than floating.
- [ ] Plant sizes feel plausible for their depth in the photograph.
- [ ] Labels do not cover the main planting composition.
- [ ] The photograph communicates the proposed future at first glance.

## G. Persistence

- [ ] Refresh the page after selecting a future.
- [ ] The selected future is restored.
- [ ] Calibration placement is restored.
- [ ] Finished state remains finished; editor handles do not reappear.
- [ ] Marker 5 remains aligned.

## H. Safety boundary

- [ ] Real AI rendering still says disabled.
- [ ] No payment, provider or backend setup is required.

## Release decision

Only mark the Plant Overlay Gate passed when every critical item above succeeds on the physical phone and the user can answer, without explanation: **“Yes, I can picture how planting would occupy this space.”**
